Design Patterns at DWP

Elements

Elements are the smallest building blocks of our page designs. The typography, fonts, colours, and layout. How we style individual parts of the user interface on the page. The large majority of Elements we use come directly from the GOV.UK elements but there are a few which are perculiar to DWP.

DWP Elements

Pages / Components

Where we combine the Elements to create pages (or parts of pages) for a single purpose. This covers the way we ask certain questions and the way we allow users to perform certain in-page tasks. Following a one-thing-per-page principle means that mainly these are page designs but they also include certain in-page components. Again we take the GOV.UK patterns as a starting point and build upon them.

DWP Page / Component Design Patterns

Interactions

Here we combine pages / components into a flow which allow the user to interact with the service. This means any place where the user is taken through a number of pages for a single purpose.

DWP Interaction Design Patterns

Services

This is where we caputure service patterns that could be used by different services we achieve the same outcomes. This covers things like what happens after a application is submitted, or how assisted digital help is offers to the users of a service.
